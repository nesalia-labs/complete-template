`DeesseJS` (brand decided 2026-06-16, repo is still `complete-template/` on disk) is a **commercial web/fullstack starter template intended to be sold**, not an open-source project. It competes in the same space as:
- supastarter
- supafast
- nextforge

**Why:** The user (founder/PM) is positioning before building. The goal is to identify what buyers want, what already exists, and what sells — *before* committing to a stack and feature set. The repo is currently empty of code by design; we're in the discovery phase.

**How to apply:**
- Treat all suggestions through the lens of "does this help sell DeesseJS" — not "is this technically elegant."
- Research questions should cover market, pricing, feature gaps, positioning, and competitor teardowns — not just tech choice.
- Don't push for a stack until the research surfaces what buyers actually pay for.
- See [[reference-template-competitors]] for the named competitor set and what to track about each.
- See [[project-positioning-hybrid-2026-06]] for the "Apple of SaaS templates" wedge.
