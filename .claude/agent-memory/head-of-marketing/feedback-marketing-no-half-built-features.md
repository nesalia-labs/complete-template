The "Apple of SaaS templates" positioning is load-bearing for the entire product (see [[project-deessejs-marketing-overview]] and head-of-product [[project-positioning-hybrid-2026-06]]). A template with 18 features done at 80% is worse than one with 8 done at 100%. **The same rule applies to marketing copy.**

**Why:** The buyer's #1 trust failure in this market is "marketing vs reality" — ShipFast's 50/100 schema audit, ixartz's "PRO plan mismatch" (users paid for the PRO plan and couldn't find the admin panel), supastarter's "what happens after you launch?" critique. Every copy claim that overpromises is a wedge gift to the next buyer's regret post.

**How to apply:**
- Never write copy that promises more than the v1 spec ships. "195 integration tests" only after they pass in CI.
- Never write "best in class" / "the most complete" without a proof link.
- Never write copy for a feature still in defer or skip status (see `documents/internal/product/features/README.md` for status legend).
- The "no fake testimonials" rule from [[reference-copy-angles]] is the same principle in social proof form.
- When the user pushes for more aggressive copy, push back with the Apple positioning rationale.
- The "after you launch" gap is the only narrative that lets us imply competitor shortcomings without saying so directly — use it.
