**Tied to the v1 build (M0-M8, ~10 weeks for 1 dev, ~6 weeks for 2):**

| Build milestone | Marketing work to run in parallel |
|---|---|
| **Pre-M5** | Pricing-page draft, lead magnet outline, buyer-comparison page draft. No public surface yet. |
| **M5 (API + SDK + CLI, weeks 7-8)** | Start State of SaaS Templates 2026 report (we have the data, 2-3 weeks writing + design). Start DeesseJS Lite extraction from v1 (3-4 weeks including landing page). |
| **M6 (AI primitives, weeks 8-9)** | Finish DeesseJS Lite. Stand up the landing page (deessejs.com) + email capture. Open the pre-launch waitlist 4-6 weeks before launch. |
| **M7 (landing page + sales, weeks 9-10)** | Public deployed demo (app.deessejs.com) goes live. 30-day money-back guarantee + 14-day trial go live. Begin beta program (10-20 design partners). Landing page copy goes live with the Stain Lu quote. |
| **M8 (pre-launch QA, week 10)** | First public artifact: one feature done to 100% on the new modular stack, deployed and linked from the landing page. The "no half-built features" proof. |
| **Launch (M8 + 1-2 weeks)** | Product Hunt launch → Indie Hackers thread → Show HN (sequenced). |
| **Launch + 4-6 weeks** | First content pillar posts ship (design-principle write-ups). |
| **Launch + 6 months** | Re-evaluate the recurring revenue question and the free/OSS tier question. |

**The demo app is the M7 flagship artifact.** It's the first public evidence that we ship what we say. It is the public proof of the "no half-built features" claim.

**Why this sequencing:**
- The lead magnets need v1 code in good shape (Lite extracts from v1, the public demo runs v1).
- The pre-launch waitlist needs landing page copy locked, which is M7 work.
- Content pillars need the public demo + the launch to reference.
- The beta program needs something to test against; M7's feature-complete state is the right time.

**Marketing risks that blow the timeline:**
- Lead magnet work starts before M5 → publishes against a moving target.
- Landing page copy lands before positioning is locked → re-work.
- Beta program starts before the wizard is stable → bad testimonials.

**How to apply:** When the user asks "what marketing work should we do this week?", match it to the current M-milestone. Pre-M5, focus on copy + pricing-page draft + lead magnet outline. M5-M6, ship the Lite + the report. M7, the landing page + waitlist + demo. Post-launch, content pillars + expansion.
