## Objection handling (agentic framing)

**"Why not supastarter?"**
→ supastarter optimizes AI for the developer (Claude Code, Cursor, Codex integration). DeesseJS optimizes the SYSTEM for AI agents. Your agents don't just help you code — they run on DeesseJS, call the tools directly, and build your product while you sleep.

**"Why not ShipFast for $199?"**
→ ShipFast is lean. You're still the developer. DeesseJS is complete — and your agents are the developers. The $100 buys you a system where you ask, it gets built.

**"Do I need to know how to prompt my agents?"**
→ You need to know what you want. The prompts and workflows are in the AI Agent Workflow Library. Your agents handle the implementation.

**"What if DeesseJS doesn't work for my project?"**
→ 14-day money-back guarantee. If your agents don't save you 3 months of development work in the first 14 days, email us for a full refund.

## Brand consistency checklist (updated)

- [ ] Calm, confident, opinionated — no hype, no superlatives
- [ ] Describes the product, doesn't oversell it
- [ ] **Agentic positioning leads** — "your agents run on DeesseJS" is the primary hook
- [ ] Agent primitives are the lead features, not a sub-section
- [ ] Never claim "agents build your entire SaaS" — honest version: "agents build features on a wired system"
- [ ] Per-tenant metering + usage-based billing are the "charge your users" payoff, not just features
- [ ] The live agent workflow demo is the visual proof — link it prominently
- [ ] Speaks to founders as architects, not developers
- [ ] "Stop hiring developers. Start asking your agents." is the emotional payoff — use it somewhere on the page
