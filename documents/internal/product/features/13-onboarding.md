# 13. Onboarding

> The first hour. 8-step wizard + persistent post-signup checklist + email sequence. The "Deleting features" flagship doc is the wedge proof.

| # | Feature | Status | Source | Note |
|---|---|---|---|---|
| 13.1 | 8-step setup wizard | ✅ | C | See [`../onboarding.md`](../onboarding.md) |
| 13.2 | Post-signup checklist (persistent) | ✅ | C | Sidebar widget, doesn't dismiss |
| 13.3 | Demo data option (sample orgs) | ✅ | D | "Skip sample data" option. Faster "first SaaS" feel. **Most templates don't ship this.** |
| 13.4 | Welcome modal (in-app) | ✅ | C | First-login banner |
| 13.5 | In-product tooltips (on key features) | ✅ | C | Optional, dismissible |
| 13.6 | Email onboarding sequence | ✅ | D | Day 0 / Day 2 / Day 7 — what to do next, with screenshots. **Sells the buyer's product to their users.** |
| 13.7 | Video tutorials | ⚪ | — | v2. Out of scope; docs cover it. |
| 13.8 | "Skip for power users" mode | ✅ | U | For repeat template buyers who know the stack |

**Notes:**
- 13.3 (demo data) and 13.6 (email sequence) are differentiators that cost little to ship.
- 13.9 (deleting features doc) is the proof that "modular" is not just marketing.
